# 📡 Resource Escalation — 资源工具缺失上报机制

## 核心理念

> **"我办不到"不是答案，"我需要 X 才能办到"才是。**

当任务推进受阻时，Agent 不能沉默跳过或编造结果。必须：明确识别缺失项 → 判定可替代性 → 结构化上报。

---

## 缺失类型分类

| 类型 | 代号 | 示例 | 可自行替代？ |
|------|------|------|-------------|
| 🧰 **工具缺失** | T | 需要 FFmpeg 但未安装、需要特定 CLI | 可尝试 `exec("winget install")` 或报错 |
| 🔑 **权限缺失** | P | 无 API key、无数据库写入权限、无文件写权限 | ❌ 不可替代 |
| 📊 **数据缺失** | D | 需要参考某个竞品数据、需要历史版本对比 | 可尝试用现有数据推导（需标注"估算"） |
| 🌐 **服务缺失** | S | 需要调用第三方 API 但未开通、服务离线 | ❌ 不可替代 |
| 🧠 **能力缺失** | C | AI 无法独立完成（如 3D 建模、音频混音、物理模拟） | ❌ 不可替代 |
| 📋 **信息缺失** | I | 需要主人确认的设计方向未定、需求模糊 | ❌ 不可替代 |

---

## 上报工作流

### Step 1: 识别缺失

当遇到以下情况时主动触发检查：
- 任务执行到某一步，工具调用返回 `command not found` / `access denied`
- 任务描述要求某种能力但当前环境明显不具备（如"渲染一段 3D 动画"）
- 需要外部数据但当前知识库无相关内容
- 需要人工决策才能继续的设计选择

### Step 2: 判定可替代性

```markdown
【替代性判定】
缺失项: <具体名称>
类型: <T/P/D/S/C/I>

可替代方案评估:
- 方案 A: <简述> — 可行/不可行（理由）
- 方案 B: <简述> — 可行/不可行（理由）

替代结论: 
- ✅ 可替代 → 记录后执行替代方案
- ❌ 不可替代 → 进入 Step 3 上报
```

### Step 3: 按优先级上报

上报按优先级 P0-P3 划分，决定响应时限和路由目标：

| 优先级 | 条件 | 响应时限 | 上报目标 | 示例 |
|--------|------|----------|----------|------|
| **P0 🔴** | 任务完全阻塞，无替代方案 | 立即 | **技术总监(TD)** 抄送 **制作人(CP)** | 缺少核心编译工具链 |
| **P1 🟠** | 任务严重降级，替代方案效果差 | 30min | **直属组长** → 30min 无响应升 TD | 缺某格式导入工具 |
| **P2 🟡** | 任务可完成但效率低 | 2h | **直属组长** | 缺批处理工具需手工替代 |
| **P3 🔵** | 信息/建议类缺失 | 24h | **知识库记录** + 抄送组长 | 需确认设计方向偏好 |

### Step 4: 填写上报单

```markdown
=== 资源缺失上报单 ===
上报时间: <ISO 8601>
上报Agent: <角色名>
任务编号: TASK-YYYYMMDD-NNN

缺失类型: <T/P/D/S/C/I>
缺失项名称: <具体缺失的工具/权限/数据/服务>

当前任务影响:
  - <任务进度: 做了哪些>
  - <阻塞节点: 卡在哪一步>
  - <影响范围: 影响哪些下游>

已尝试的替代方案:
  1. <方案A> — 结果: 失败/效果差/无权限/…
  2. <方案B> — 结果: <同上>

建议解决方案:
  - <建议1: 安装/购买/申请权限/找替代工具>
  - <建议2: 如不可行，是否调整任务范围>

紧急程度: P<P0-P3>
期望回复: <需安装/需批准/需指示/需采购决策>
```

### Step 5: 跟踪闭环

上报后追踪以下状态：

```markdown
| 上报ID | 时间 | Agent | 类型 | 缺失项 | 优先级 | 状态 | 处理人 | 解决方案 |
|-------|------|-------|------|--------|--------|------|--------|----------|
| ESC-001 | 06-03 | TD | T | Unity IL2CPP 编译链 | P0 | ✅已解决 | CP | 安装 Windows SDK |
| ESC-002 | 06-04 | DL | P | 代码仓库推送权限 | P1 | ⏳处理中 | QD | 待开通 |
```

状态流转：
```
新建 → 已受理 → 处理中 → 已解决
  ↓        ↓
过期(超时)  无法解决(转为架构决策)
```

## 上报记录管理

每次上报必须记录到工作区的资源缺失日志：

```markdown
# 📡 资源缺失日志

| 编号 | 日期 | Agent | 类型 | 缺失项 | 优先级 | 状态 | 处理人 |
|------|------|-------|------|--------|--------|------|--------|
```

文件路径：**`/Logs/resource-gaps.md`**（统一位置，所有 Agent 可读写）

记录格式：
```markdown
| ESC-<NNN> | <YYYY-MM-DD> | <角色> | <T/P/D/S/C/I> | <缺失项> | P<N> | 🟢已解决/🟡处理中/🔴待解决 | <处理人>
```

## 防守原则

### 必须上报的场景
- ✅ 工具返回 `command not found`
- ✅ API 返回 401/403/500 且无法绕过
- ✅ LLM 明确表示无法执行的操作（如"我不能帮您代发邮件"）
- ✅ 需要人类判断的决策点（"选方案A还是B？"）
- ✅ 超过 3 次尝试仍无法解决的技术问题

### 禁止行为
- ❌ 没有替代方案却编造结果继续推进
- ❌ 遇到权限拒绝后尝试绕过（用其他账号/硬编码密码）
- ❌ 同一问题反复上报却不更新上报状态
- ❌ P0 上报不抄送 CP

### 超时自动升级规则
| 优先级 | 超时 | 升级路径 |
|--------|------|----------|
| P0 | 15min 无响应 | 直接联系制作人 |
| P1 | 30min 无响应 | 升 TD/QD |
| P2 | 2h 无响应 | 升 TD/QD，抄送 Auditor |
| P3 | 24h 无响应 | 升直属组长 |

## 附录：实战参考场景（2026-06-03 CF.Game 集群搭建）

以下是在本集群搭建中实际遇到的资源缺失场景，供 Agent 参考如何判定和上报：

### 场景 1：CLI 工具缺失（T 类）

```
上报: exec("multica agent update ...") 执行失败
      → command not found: multica
判定: 工具缺失（multica CLI 未在 PATH 中）
替代: 尝试 $env:PATH += ";C:\Users\lixin\.multica\bin"
结果: ✅ 可替代，无需上报
```

### 场景 2：API 路由设计不清（I 类）

```
上报: curl PATCH /api/agents/<id> 返回 400
      → 需要 workspace_id 但我没传对你的路由要求
判定: 信息缺失（API 路由规则不明确）
替代: 能否通过 header / query / body 传递？逐一尝试
结果: ✅ 发现 query param 可解，无需上报
```

### 场景 3：Python stdout 编码不兼容（T 类）

```
上报: subprocess.run() 捕获 multica CLI 输出
      → UnicodeDecodeError: 'gbk' codec can't decode byte
判定: 编码兼容问题（CLI 输出含中文，Python 默认 GBK 解码失败）
替代: 设 encoding='utf-8', errors='replace'
结果: ✅ 可替代，记录为已知经验
```

### 场景 4：运行时缺少某工具链（T 类，上报示例）

```
上报: exec("python3 import_role_skills.py") 返回 9009
      → 系统找不到 python3
判定: 工具缺失（windows 下 Python 命令名不是 python3 而是 python）
替代: 尝试 python, py, 或 $env:PATH 检查
结果: ✅ 发现命令名为 python，无需上报
```

### 场景 5：需要人类审批的能力缺失（C 类 — 必须上报）

```
上报: Agent 需要创建新的工作流级别技能
      → 此为 L1 操作，不在任何 Agent 权限范围内
判定: 能力缺失 + 权限缺失
替代: 无替代。必须上报制作人 CP 审批
路由: P1 → TD 抄送 CP
```

### 关键教训

| 发现 | 归类 | 沉淀位置 |
|------|------|----------|
| PowerShell 中文编码问题频繁 | 环境缺陷 | knowledge-precipitator → 编码避坑 |
| Multica CLI 参数名不统一（--content-file vs --file） | UI/文档问题 | resource-gaps.md → 待产品改进 |
| 批量操作无事务回滚 | 产品限制 | operaion-patterns → 操作前必须备份 |
| ConvertFrom-Json 对中文 JSON 不支持 | 环境缺陷 | agent-file-update → 附录避坑表 |
