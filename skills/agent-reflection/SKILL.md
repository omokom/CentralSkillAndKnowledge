# 🔄 Agent Reflection — 任务反思与自我提升

## 职责

每次任务完成后执行结构化反思，提取经验教训，驱动自我成长。

## 反思时机

| 时机 | 触发条件 | 深度 |
|------|----------|------|
| 快速反思 | 每次任务交付后 5min 内 | 浅（5min） |
| 深度反思 | 任务失败、返工、耗时 >2h | 深（15min） |
| 每日总结 | 当天最后任务完成后 | 中（10min） |
| 周度回顾 | 每周最后一项任务 | 深（30min） |

## 反思工作流

### 1. 收集任务事实
```
read("task_log.md")  // 任务执行日志
read("回顾当前任务的输入输出物")
```

### 2. 回答五个反思问题

#### Q1: 这次任务做对了什么？
- 哪个决策最有效？
- 哪个步骤特别顺畅？
- 用了什么好方法可以复用？

#### Q2: 这次任务做错了什么？
- 哪里走了弯路？
- 哪个判断是错的？
- 什么资源/信息本应提前准备？

#### Q3: 如果重来一次会怎么做？
- 第一步做什么改变？
- 什么信息需要提前获取？
- 什么环节可以跳过/简化？

#### Q4: 学到了什么新东西？
- 新工具/新API/新方法？
- 新的排查思路？
- 某配置参数的正确含义？

#### Q5: 谁可能需要知道这个？
- 同组其他角色？
- 上游/下游角色？
- 存档到知识库？

### 3. 生成反思笔记

```markdown
---
type: reflection
date: YYYY-MM-DD
task: TASK-xxxx
role: <你的角色>
---

# 反思: <任务主题>

## ✅ 做得好的
- <条目1>
- <条目2>

## ❌ 可以改进
- <条目1> → 改进方案: <xxx>
- <条目2> → 改进方案: <xxx>

## 🧠 新学到
- <条目1> (已沉淀到知识库: knowledge/ref/xxx.md)

## 📌 后续行动
- [ ] <待办项1>
- [ ] <待办项2>
```

### 4. 触发知识沉淀
如有值得沉淀的经验，调用 **knowledge-precipitator** 写入知识库。

## 反思输出要求

- [ ] 每个反思至少 1 条"做得好的"和 1 条"可以改进"
- [ ] 改进项必须有具体的改进方案，不能只列问题
- [ ] 新知识必须标注是否已沉淀
- [ ] 反思笔记存档到 `reflections/` 目录
- [ ] 每周至少 1 次深度反思

## 防守原则

- 反思是为了成长，不是自责。用"学到了"代替"真蠢"
- 不过度反思：常规任务只做快速反思（3-5min）
- 反思不代替执行：反思后如有待办，继续处理而非停在这个环节
